# API Reference

Everything is a named export from the package root:

```ts
import {
  diff, patch, diffWith,           // functions
  serialize, deserialize,          // diff wire format
  stringify, parse,                // general-purpose value codec
  DELETED, ADDED, CHANGED, INSERTED, REMOVED, // op-type constants
} from "@opentf/obj-diff";
```

| Export | Kind | Summary |
| :--- | :--- | :--- |
| [`diff`](#diff) | function | Deep-compare two values into a list of operations. |
| [`patch`](#patch) | function | Apply a diff to reconstruct the target. |
| [`diffWith`](#diffwith) | function | `diff` with a custom equality function. |
| [`serialize`](#serialize) | function | Encode a diff to a self-describing JSON string. |
| [`deserialize`](#deserialize) | function | Decode a serialized diff back into live typed values. |
| [`stringify`](#stringify) | function | Encode **any** value to a type-safe JSON string. |
| [`parse`](#parse) | function | Decode a `stringify` string back into live typed values. |
| [`DiffResult`](#diffresult-type) | type | The shape of a single operation. |
| [Op constants](#op-type-constants) | const | `DELETED`, `ADDED`, `CHANGED`, `INSERTED`, `REMOVED`. |

## `diff`

```ts
function diff(a: unknown, b: unknown): DiffResult[]
```

Performs a deep comparison and returns the list of operations that transform `a` into `b`. An empty array means the two values are deeply equal.

**Parameters**

| Name | Type | Description |
| :--- | :--- | :--- |
| `a` | `unknown` | The source (original) value. |
| `b` | `unknown` | The target value to compare against. |

**Returns** — `DiffResult[]`, in application order.

```ts
diff({ a: 1 }, { a: 2 });
//=> [{ type: 2, path: ["a"], value: 2 }]
```

### Compact array diffs

`diff` trims the common prefix and suffix of arrays and resolves the remaining middle with a bounded shortest-edit-script (Myers LCS) pass. Inserting one element at the front of a 10,000-element array produces **1 op**, not 10,001:

```ts
diff([1, 2, 3], [0, 1, 2, 3]);
//=> [{ type: 3, path: [0], value: 0 }]

diff([1, 2, 3, 4], [1, 4]);
//=> [{ type: 4, path: [1] }, { type: 4, path: [1] }]
```

`INSERTED` / `REMOVED` indexes are **application-time** (RFC 6902 style): apply the ops in order and each index is valid at the moment it is applied. When two arrays are too different (an internal edit-distance cap is exceeded), `diff` falls back to index-by-index comparison to bound worst-case time and memory.

## `patch`

```ts
function patch(obj: unknown, patches: DiffResult[]): unknown
```

Applies a diff to `obj` and returns the reconstructed value. The round-trip always holds: `patch(a, diff(a, b))` is deep-equal to `b`.

**Parameters**

| Name | Type | Description |
| :--- | :--- | :--- |
| `obj` | `unknown` | The source value to apply the diff to. |
| `patches` | `DiffResult[]` | A diff produced by `diff` or `diffWith`. |

**Returns** — the patched value.

```ts
const a = { a: 1 };
const b = { a: 2 };

patch(a, diff(a, b)); //=> { a: 2 }
```

<Callout type="note">

Added and changed values are placed into the result **by reference**, not cloned. See [Caveats](/docs/caveats) for aliasing details.

</Callout>

## `diffWith`

```ts
function diffWith(
  a: unknown,
  b: unknown,
  isChanged: (a: object, b: object) => boolean | undefined,
): DiffResult[]
```

Like [`diff`](#diff), but you supply a comparator to override equality for specific object types (e.g. MongoDB `ObjectId`, custom classes).

**Parameters**

| Name | Type | Description |
| :--- | :--- | :--- |
| `a` | `unknown` | The source value. |
| `b` | `unknown` | The target value. |
| `isChanged` | `(a, b) => boolean \| undefined` | Return `true`/`false` to force the result, or `undefined` to fall back to the built-in comparison. Invoked **only when both values are objects**. |

**Returns** — `DiffResult[]`.

```ts
import { diffWith } from "@opentf/obj-diff";
import { ObjectId } from "bson";

diffWith(recordA, recordB, (a, b) => {
  if (a instanceof ObjectId && b instanceof ObjectId) {
    return a.toString() !== b.toString();
  }
});
```

## `serialize`

```ts
function serialize(diff: DiffResult[]): string
```

Encodes a diff into a self-describing JSON string that survives transport. A diff holds live values by reference, so `JSON.stringify` loses their types; `serialize` preserves them. See [Serialization](/docs/serialization) for the format and rationale.

**Parameters**

| Name | Type | Description |
| :--- | :--- | :--- |
| `diff` | `DiffResult[]` | A diff produced by `diff` or `diffWith`. |

**Returns** — a JSON `string`. Preserves native types (`Date`, `Map`, `Set`, `TypedArray`, `URL`, …) plus circular references and shared identity among plain objects/arrays. Throws on symbols, functions, class-instance values, and cycles that run through a `Map`/`Set`/`Error`.

```ts
const wire = serialize(diff(a, b));
```

---

## `deserialize`

```ts
function deserialize(wire: string): DiffResult[]
```

Decodes a string from [`serialize`](#serialize) back into a diff whose values are live, correctly-typed objects — ready for [`patch`](#patch).

**Parameters**

| Name | Type | Description |
| :--- | :--- | :--- |
| `wire` | `string` | The output of `serialize`. |

**Returns** — `DiffResult[]` with reconstructed values.

```ts
patch(a, deserialize(wire)); // Date / Map / Set / … restored exactly
```

<Callout type="note" title="Temporal at decode time">

Deserializing a `Temporal` value requires a `Temporal` implementation on `globalThis` (native or a globally-installed polyfill). This applies to `parse` too. Encoding never needs one.

</Callout>

## `stringify`

```ts
function stringify(value: unknown): string
```

The general-purpose counterpart to [`serialize`](#serialize): encodes **any** JavaScript value — not just a diff — into a self-describing, type-safe JSON string. Uses the same codec, so it preserves every supported native type (`Date`, `Map`, `Set`, `TypedArray`, `ArrayBuffer`, `DataView`, `Error`, `URL`, `BigInt`, `Temporal`, `NaN`/`±Infinity`/`-0`, `undefined`, …) plus circular references and shared identity among plain objects/arrays. A type-preserving drop-in for `JSON.stringify`. See the [superjson comparison](/docs/comparison#type-safe-serialization-vs-superjson).

**Parameters**

| Name | Type | Description |
| :--- | :--- | :--- |
| `value` | `unknown` | Any serializable JavaScript value. |

**Returns** — a JSON `string`. Throws on symbols, functions, and class-instance values.

```ts
const wire = stringify({ when: new Date(), tags: new Set([1, 2]), big: 42n });
```

---

## `parse`

```ts
function parse<T = unknown>(wire: string): T
```

Decodes a string from [`stringify`](#stringify) back into the original value with its types restored — the type-preserving counterpart to `JSON.parse`.

**Parameters**

| Name | Type | Description |
| :--- | :--- | :--- |
| `wire` | `string` | The output of `stringify`. |

**Returns** — the reconstructed value (typed as `T`).

```ts
const value = parse(wire); // Date / Set / BigInt restored
```

## `DiffResult` type

```ts
type DiffResult = {
  type: 0 | 1 | 2 | 3 | 4; // see the op-type table below
  path: Array<unknown>;    // path to the property; Map entries use the Map key itself
  value?: unknown;         // the new value (present for ADDED, CHANGED, INSERTED)
};
```

### Op-type constants

| Value | Constant | Meaning | Applies to |
| :---: | :--- | :--- | :--- |
| `0` | `DELETED` | Key no longer exists (`delete obj.k` / `map.delete(k)`; on an array index it leaves a hole). | objects, Maps, arrays (sparse) |
| `1` | `ADDED` | New key/value (on arrays: assign at index). | objects, Maps, Sets, arrays |
| `2` | `CHANGED` | Value replaced at the path. | all |
| `3` | `INSERTED` | Insert `value` at the index, shifting later elements right (`splice(i, 0, v)`). | arrays only |
| `4` | `REMOVED` | Remove the element at the index, shifting later elements left (`splice(i, 1)`). | arrays only |

```ts
import { DELETED, ADDED, CHANGED, INSERTED, REMOVED } from "@opentf/obj-diff";

diff(a, b).filter((op) => op.type === REMOVED);
```

An empty `path` (`[]`) denotes the **root** — the entire source was replaced by the target (e.g. comparing an object to `null`).
